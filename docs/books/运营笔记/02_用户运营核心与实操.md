# 第三部分 用户运营核心 + 第四部分 用户运营实操

## 🎯 核心主张

**用户运营的本质是分层经营：把80%的精力投在金字塔顶端20%的用户身上，让顶端用户带动底层用户**——你的价值不取决于平台总用户数，而取决于"有多少用户认识你、信任你、愿意为你做事"。

---

## 📌 关键框架

### 框架一：用户金字塔模型（全书最核心的框架）

```
       ▲
      ╱ ╲    第一级：运营人员（你自己）
     ╱   ╲   ← 你的能力决定下面所有人的上限
    ╱─────╲
   ╱       ╲  第二级：用户管理工具
  ╱  版主体系 ╲  （版主、社群、QQ群、用户自组织）
 ╱  社群联盟  ╲
╱─────────────╲
   第三级：有价值的用户
  （活跃产出UGC的核心用户，<20%）
─────────────────
        第四级：一般性用户
        （80%的潜水用户）
```

**三条铁律**：
1. **80/20法则**：用80%时间服务顶端20%用户
2. **能力上限**：你打85分，用户最高也只能打85分（"将熊熊一窝"）
3. **互相影响**：上层影响下层，文化由20%核心用户带动80%

**倒置应用——用户漏斗**：把金字塔反过来用做产品需求筛选——核心功能优先满足顶端用户，越往下越是细枝末节。

### 框架二：威逼利诱——用户关系处理四字诀

| 字 | 含义 | 操作落点 |
|----|------|---------|
| **威** | 树立威信 | 社区规则 + 解决问题 + 贴近用户 |
| **逼** | 自我倒逼 | 先逼自己提升业务能力，规则给用户敬畏，本质是「帮用户成长」 |
| **利** | 给好处 | 权限（循序渐进给）+ 小恩小惠（虚拟物品比钱更动心）+ 内容曝光 |
| **诱** | 诱惑+诱导 | 等级体系/活动诱惑 + 优质用户帮你引导回复氛围 |

**作者强调**：威信 ≠ 威胁，运营和用户在某种程度上应该是平等的。

### 框架三：灵魂用户的5个特征

通过「魔女花茶」案例（猫扑女神，2006-2014连续8年每天更新帖子，累计回复25万、点击破亿）总结：

1. **超高忠诚度**（时间维度极长）
2. **形象正面**（不靠脸蛋吃饭，有现实身份背书）
3. **零功利心**（不在帖子里给自己宣传）
4. **平易近人**（聚合优质粉丝氛围，自我治理）
5. **媒体认可**（具备正向传播价值）

**社区"双灵魂"原则**：必须同时有官方灵魂（把社区当儿子养的运营人）+ 用户灵魂（社区代言人）。

### 框架四：拉用户的6字真经

**用心、坚持、专注**——作者说所有成为高手的运营，做的都是这6个字。

具体操作5步：
1. 带着目的去沟通（明确这个用户能做什么）
2. 投其所好（像谈恋爱一样找切入点）
3. 落地（让用户在平台上产生价值，不只拿联系方式）
4. 质量大于数量
5. 日常维护（不只在用得着时找人）

### 框架五：无预算做活动的6条经验

1. 平时积累用户资源（找网红/明星做评委）
2. 牢记活动目的（宣传品牌？带来收入？）
3. 解决奖品问题：找赞助 / 联合销售部门 / 找领导（下下策）
4. 文案细节：标题 + 内容（简介/流程/奖项/注意事项）+ 奖品图必有
5. 灵活性（如"健康美腿大赛"加"健康"二字规避审核风险）
6. 主动运营+站外推广+利用站内资源

### 框架六：收集癖驱动的用户激励体系

> 在社区里，用户最想要的一定是现实生活中得不到的东西。

用户收集的不是物品，而是：**存在感 + 荣誉感 + 别人和自我的认可感**

设计原则：
- 等级不能太少（永远有下一档）
- 勋章/虚拟物品要多样
- 周边产品成本极低但效果好（小扑38元成本被炒到800元）
- 钱不是万能（对高净值用户，奖金不到10万级就是俗套）

## 🔧 可落地的交付物

### 交付物2：威逼利诱4字应用清单（运营客户/粉丝时直接照搬）

**威**——
- [ ] 你有明确的服务规则吗？（比如"凌晨不接非紧急工单"）
- [ ] 客户问题响应SLA是否清晰？
- [ ] 你能不能不卑不亢地说"这个不能改"？

**逼**——
- [ ] 你的业务能力是否在持续升级？（如果不升级，凭什么客户跟你走？）
- [ ] 客户的不合理要求，你能否帮他重新认识真正的需求？

**利**——
- [ ] 你的客户能从你这里拿到什么"现实里拿不到"的东西？（行业洞察？牵线？）
- [ ] 老客户是否有差异化权益？

**诱**——
- [ ] 你有没有为顶层客户设计"够得着的下一档"？
- [ ] 你的内容/活动里有没有"引导员"在带氛围？

### 交付物3：灵魂用户识别checklist

下次面对一个客户/粉丝，对照5条判断他是不是潜在的"灵魂用户"：

| 维度 | 判断点 |
|------|--------|
| 忠诚度 | 跟你超过2年？反复购买/反复出现？ |
| 形象 | 现实身份能背书你的品牌？ |
| 功利心 | 跟你互动时不直接索取利益？ |
| 亲和力 | 能在你周围聚合其他用户？ |
| 媒体价值 | 故事有传播性？可以推到媒体？ |

**5项中4项以上**→ 这就是你要重点经营的灵魂用户。

### 交付物4：活动文案速查表

每次做活动前对照检查：
- [ ] 标题：15-32字，关键词钩子≥4个，加（图）（视频）标注
- [ ] 简介：3行内说清是什么、为什么、给什么
- [ ] 流程：步骤清晰，不超过5步
- [ ] 奖项：必须有图，必须有参与奖兜底
- [ ] 站外推广方案：3个渠道以上
- [ ] 引导员：至少3个核心用户先回复带氛围

### 交付物5：用户运营的6个易犯错误自查表

每月看一遍：
- [ ] 是否被动等用户上门，没去"站外"找？
- [ ] 是否忽视了晚上这个高价值时间段？
- [ ] 是否对用户简单粗暴（没耐心解释）？
- [ ] 是否过度依赖文字沟通（QQ/微信），不会用电话/面谈？
- [ ] 当用户群体打架时，是不是去和稀泥而不是按规则裁决？
- [ ] 是不是把本该自己做的工作偷懒丢给用户？

## 💡 重要洞见与案例

### 洞见1：「最好的用户都是培养出来的，绝对不是等来的」

养鱼VS钓鱼的比喻：
- 钓鱼者：只会让池塘的鱼越来越少
- 渔夫：清楚池塘里适合养什么鱼，定期撒鱼苗

**用户维护策略一句话**：重点培养，广撒网。

### 洞见2：用户调研不可迷信

> 一个从来不接触用户的产品人员，是不配做产品的；一个从来不接触用户的运营人员，只能算是入门的初级运营人员。

**作者直接打脸调研主义**：用户跟你不熟，对方根本不会跟你说实话。最客观的需求来自长期接触用户后建立的情感连接。

### 洞见3：「初期自己做、中期一起做、后期交给用户」——放权三段式

放权不是一开始就甩锅，而是：
- **初期**：自己勤加练习，吃透流程
- **中期**：和用户一起做，培养接班人
- **后期**：完全交给用户，自己抓宏观

> 一个没有"用户禁卫军"的运营，哪怕你单兵作战能力再强，你永远都只是一个普通运营。

### 洞见4：「魔女花茶」灵魂用户案例——8年每天更新

猫扑女神花茶，从2006年到2014年，每天更新帖子。8年×365天 = 近3000条更新。这种"长期主义"的灵魂用户：
- 不靠脸吃饭（白领美女，有投资理财能力，开健身房+做顾问）
- 上过《天天向上》、《幸福晚点名》（媒体二次背书）
- 主动要求"尽量少为她推荐"（反过来要求节制）

**作者的运营动作**：
- 每次更新照片必推黄金位
- 每条回复都看
- 连续3年为她做活动（成本0，效果爆炸）

### 洞见5：周边产品的经济账

猫扑小扑玩偶：
- 14厘米普通版2008年成本10元 × 1000只 = 1万元
- 可以支撑整年活动 + 版主体系运转
- 30厘米版成本38元，在淘宝被炒到800元

**关键策略**：周边发放要保持稀缺性
1. 新品先在核心用户里发放（不是所有人都有）
2. 以活动形式发放，控制数量
3. 等核心用户都有了再放量、开放售卖
4. 同时下一款新品已经做出来了

### 洞见6：用户美女维护——本质是「区别对待 + 重视细节」

虽然这是个特定群体案例，但本质方法可以迁移：
- 不要在需要时才想起人家（平时嘘寒问暖、生日祝福）
- 任何时候接招（高净值用户的时间不规律）
- 主动学习对方需要的语言/语气（作者甚至模仿女孩语气教对方改标题）

### 洞见7：猫扑大杂烩案例——把贴贴2年做到反超主版

- 接手时贴贴只是大杂烩流量的1/10到1/20
- 2年后发回帖涨20倍，PV涨30倍

四个时间节点：
1. 用80/20抓重点版（不平均用力）
2. 抓版主管理（用户管理工具的核心）
3. 抓联盟管理（用户自组织，贡献60-70%数据）
4. 抓无线运营（看似与KPI无关，但因为接触了就主动学，后来全权接管）

> 你肯给自己机会，别人就一定会给你机会。

## 📖 值得记录的金句

> 你的价值取决于你手上有多少可支配的用户，而不是平台总体的用户数量。

> 与其每天挖空心思琢磨用户、"认为"用户需要什么，不如真心实意地去和用户沟通交流。用户运营的技巧在于以情动情。

> 兵熊熊一个，将熊熊一窝。如果你自己都没把工作理解到位，就把工作直接交给用户去做了——想想这样做的后果是什么！

> 在虚拟世界里，用户最需要的是他在现实里无法得到的东西。

> 在用户面前，我们永远都是学生、是小白。如果你真正把产品当成自己的孩子，你就会觉得一直亏欠你的用户。

> 用户是我的老师，他们教给我的远远比我的领导教给我的要多很多。

> 你最开始的起点应该是把最近在手边的产品当作竞争对手，而不是像某些战略家一样，自己尚且青涩就把行业龙头当对手。

> 我们可以通过产品手段和运营手段刺激用户，让用户获得更多的满足感——他们收集的则是：存在感、荣誉感、别人和自我的认可感。
