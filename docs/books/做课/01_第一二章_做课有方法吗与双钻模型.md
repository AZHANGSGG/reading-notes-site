# 第一二章：做课有方法吗 & 双钻模型

> **本部分核心**：做课不是凭感觉，而是一套可以被学习、复制的结构化方法。双钻模型是全书的骨架。

---

## 第一章：做课，有方法吗

### 🎯 核心主张
课程设计不是玄学而是科学——用产品经理的结构化思维，任何有3年以上经验的职场人都可以做出80分以上的好课。

### 📌 关键框架

**知识资产的三个层次**（决定课程类型和价值）：

| 层次 | 描述 | 适合做什么 |
|------|------|-----------|
| Level 1：重要项目经验 | 以负责人身份参与完整项目，总结出特定方法 | 知识小课 |
| Level 2：重复工作方法论 | 多次重复做，形成稳定可依赖的方法论 | 知识小课 / 体系课程 |
| Level 3：直觉判断决策 | 在某领域形成下意识判断能力，靠直觉就能做结论 | 体系课程，最高价值 |

**学员在学习时面临的四道关卡**：
1. **不想学**：学习是反人性的，免费课完课率是付费课的1/5
2. **学不完**：没耐心看完，热播剧开播时学习平台活跃度直线下降
3. **听不懂**：讲师讲得慷慨激昂，学员却不知道学到了什么
4. **用不了**：感觉听了个热闹，回到工作中不知道怎么用

### 🔧 可落地的交付物

**知识资产自盘点表**（马上填，近3年的工作经历）：

| 工作内容 | 类型（项目/重复/直觉）| 完成时间 | 成果 | 参与度 | 给别人讲过？ | 优先级 |
|---------|-------------------|---------|------|--------|------------|--------|
| | | | | | | |
| | | | | | | |

优先级评分维度：时间（越近越高）、成果（影响越大越高）、参与度（全程负责最高）、可用性（给人讲过的优先）、稀缺性（有创新优先）。

**四道关卡的内容自检**（发布内容前对照）：
- [ ] 第一关：开头30秒内，有没有让目标受众感受到「这跟我有关」？
- [ ] 第二关：内容节奏是否有张有弛，没有持续堆砌知识点？
- [ ] 第三关：核心方法是否讲到了「学员可以按方法实践」的颗粒度？
- [ ] 第四关：结尾是否告诉了学员「这个方法具体怎么用在你的工作里」？

### 💡 重要洞见
好的课程不只是「给知识」，而是「带着学员走完整个学习过程」——这意味着课程设计不只是内容设计，更是**用户体验设计**。

## 第二章：双钻模型——做课的四步流程

### 🎯 核心主张
做课有一套「双钻模型」，四步走完就能做出质量有保证的课程——发散→收敛→发散→收敛，每步都可以反复调整，不是线性单向的。

### 📌 关键框架

```
盘点知识资产（发散）→ 设定教学目标（收敛）→ 确定课程结构（发散）→ 设计学习体验（收敛）
```

**第一步：盘点知识资产**
- 整理近3年工作经历，按重要项目/重复工作/直觉判断分类
- 用5个维度打分排序（时间/成果/参与度/可用性/稀缺性）
- 输出：值得做课的方向清单

**第二步：设定教学目标**
目标学员四层级：`没概念 → 有了解 → 有经验 → 资深`（知识小课只针对一类）

学习成果四程度：

| 程度 | 含义 | 适合场景 |
|------|------|---------|
| 基本知道 | 了解概念原理 | 新知新见类课程 |
| 深刻理解 | 理解每个步骤 | 工作方法类（认知层）|
| 简单应用 | 跟着步骤做出来 | 工作方法类（实操层）|
| 熟练掌握 | 实战训练后内化 | 体系课程 |

**第三步：确定课程结构**
核心路径：`工作经验 → 萃取方法论 → 拆解为知识点 → 打磨成交付物`

交付物（工具/模板/清单）是课程真正的价值载体，是学员「拿来即用」的东西。

**第四步：设计学习体验**
截屏点「1234」：
- 1清单：总结要注意的因素
- 2对比：加深认知的对比表格
- 3支柱：维恩图/三角形展示关系
- 4象限：两维度判断决策工具

### 🔧 可落地的交付物

**做课启动前决策框架**（动手之前先回答这6个问题）：

1. **我的知识资产在哪个层次？**（项目/重复/直觉）→ 决定做知识小课还是体系课程
2. **主要对哪类学员说话？**（没概念/有了解/有经验/资深）→ 只选一类
3. **我要交付什么程度的成果？**（知道/理解/应用/掌握）→ 决定课程深度
4. **我的课程里有什么可以直接拿来用的交付物？**（工具/清单/模板）→ 决定课程价值
5. **学员会在哪个关卡流失？**（不想学/学不完/听不懂/用不了）→ 决定设计重点
6. **这门课做出来，我敢让别人说「这不如XX课」吗？**→ 决定是否真的值得做

**笔记→课程的萃取检验**：
把你打算讲的内容，问自己：「如果只能给学员一样东西，这一章能让他拿走什么具体的工具或标准？」如果只能说「框架很好」，说明还没萃取到位，需要继续往下挖。

### 💡 重要洞见
「讲师要想将自己的工作经验传输给对方作为可用的知识，需要一个转换过程。」——经验不等于知识，经验需要被萃取成方法论，方法论需要被打磨成交付物，交付物才是学员真正能用的东西。

## 第一二章总结

| 步骤 | 核心问题 | 交付物 | 最常见错误 |
|------|---------|--------|-----------|
| 盘点知识资产 | 我有什么能讲 | 课程方向清单 | 低估自己的经验价值 |
| 设定教学目标 | 给谁解决什么问题 | 目标学员+成果定义 | 想覆盖所有人 |
| 确定课程结构 | 怎么组织知识点 | 课程大纲+交付物清单 | 直接讲经验，没有萃取 |
| 设计学习体验 | 怎么让学员走完全程 | 截屏点设计（1234）| 内容好但设计差 |
