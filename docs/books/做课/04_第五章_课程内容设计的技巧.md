# 第五章：课程内容设计的技巧

> **本章核心**：内容好但呈现差，等于白费。这是「最后一公里」——每个细节都在影响学员是否真正完成学习、感觉有收获。

---

## 🎯 核心主张
新职业类课程的学员需求多元、变化快——讲师掌握了这些呈现技巧，就能在内容质量相同的情况下，让学员的收获感大幅提升。

---

## 📌 关键框架

### 个人介绍：两个原则

- **适度**：如实描述，不夸大，平等对话，切忌居高临下
- **相关**：只讲与本课内容相关的经历，其余一笔带过

### 讲好概念的三层递进

| 层次 | 做法 | 避免 |
|------|------|------|
| 第一层 | 用案例展示，让学员先看到「这是什么样子的」 | 直接念百度百科式定义 |
| 第二层 | 在案例基础上再讲维度和原理 | 一上来就讲抽象框架 |
| 第三层 | 把颗粒度挖到学员可以按方法实践的程度 | 停在「大方向对了」就收手 |

### 讲重点的四种升级方式

| 错误做法 | 正确做法 |
|---------|---------|
| 一页PPT塞满内容 | 把一页拆成多张，每步单独成页 |
| 信息混杂，逻辑不清 | 用表格把逻辑关系拆分清楚 |
| 步骤只是罗列文字 | 把步骤图示化，可视化呈现 |
| 文字堆砌 | 信息结构化，标注重点 |

### 案例设计：三种类型

| 类型 | 价值 | 用法 |
|------|------|------|
| 我操盘过的 | 最高，有细节有温度 | 核心案例，深挖细节 |
| 我参与过的 | 次之，可讲失败经验 | 替补，谨慎使用 |
| 我看到的 | 辅助，增加场景感 | 非核心，切忌臆想 |

### 讲课语气：口播腔 vs 解说腔

| ❌ 解说腔 | ✅ 口播腔 |
|---------|---------|
| 「数字很有意思」 | 「有个数字挺奇怪的」 |
| 「值得注意的是」 | 「有一点要说一下」 |
| 「综合来看」 | 「说到底」 |

### 峰终定律：结尾决定记忆

学员印象最深的是课程的「顶峰」和「结尾」。结尾不是重复目录，而是：
- 带着学员回顾**核心方法**的完整路径
- 用**一张图**总结（值得截图保存的截屏点）
- 补充方法在**突发情况**下的适配性或更多应用场景

## 🔧 可落地的交付物

### 个人介绍模板（适度+相关原则）

```
「大家好，我是[姓名]。

过去[X]年，我在[具体领域]做过[具体事情]，
其中[与本课最相关的经历1句话]。

今天这门课，我重点分享[核心方法]，
学完之后你可以[具体能做什么]。

好，我们开始。」
```

规则：
- 个人介绍不超过2分钟
- 只讲与本课直接相关的经历
- 不逐字读PPT，把经历展开讲，而非罗列

### 录制前Checklist

**环境**：
- [ ] 安静封闭（杂物房/厨房比大会议室更好，回音小）
- [ ] 话筒测试完成（先试讲30秒，回听确认无杂音、无喷麦）

**内容准备**：
- [ ] 完整浏览一遍PPT
- [ ] 逐字稿熟悉，知道每页重点在哪
- [ ] 开头和结尾各留3秒空白（方便后期剪辑）
- [ ] 段落间停顿1-2秒

**录制规范**：
- [ ] 语速目标：每分钟150-250字
- [ ] 说「你」不说「大家」「你们」
- [ ] 出现中断：停顿几秒说「再来一次」，从断掉那句句首重录，不用全部重来
- [ ] 音频双备份（录音笔+手机或电脑同时录）

### 案例设计SCQA模板（引入型案例用）

```
情景（Situation）：
描述事情发生的时间、地点、背景——用学员熟悉的工作场景

冲突（Complication）：
实际发生了什么，和预期有什么落差——这是吸引人继续看的关键

疑问（Question）：
学员心里自然会产生的那个问题——「那怎么办？」「为什么会这样？」

答案（Answer）：
给出要讲的知识点或交付物——简洁，让学员一眼看到
```

示例填写（以「如何设计销转模型」为例）：
```
情景：你负责一门在线课程的销售转化，上个月转化率只有2%
冲突：你加大了推广力度，花了更多钱，转化率依然没有提升
疑问：钱花了，流量有了，为什么就是不转化？
答案：问题出在销转模型的设计——今天我们来讲清楚怎么搭
```

### PPT审核标准快速版

**内容层**：
- [ ] 每页能直接看出重点，不需要听讲就能理解主旨
- [ ] 每张内容页有截屏点（清单/对比/支柱/象限之一）
- [ ] 一页内容不超过7个元素
- [ ] 目录层级不超过2级

**规范层**：
- [ ] 无错别字
- [ ] 图片清晰不变形
- [ ] 使用Office自带字体（避免版权问题和乱码）
- [ ] 未植入过多广告/二维码/社交媒体链接

**脱敏层**：
- [ ] 业务数据已做模糊化处理
- [ ] 案例里没有未经授权的真实个人信息

### 讲课口播腔 vs 解说腔对照表

| 情景 | ❌ 解说腔 | ✅ 口播腔 |
|------|---------|---------|
| 引出数字 | 「数字很有意思」 | 「有个数字挺奇怪的」 |
| 转换话题 | 「与此同时」 | 「就在这个时候」 |
| 做总结 | 「综合来看」 | 「说到底」 |
| 提示重点 | 「值得注意的是」 | 「有一点要说一下」 |
| 推进节奏 | 「第一点……第二点……」 | 「先说这个，再说那个」 |
| 回顾上文 | 「如上所述」 | 「刚才说的是……」 |

**写完之后，大声念一遍**——念起来别扭的句子，全部重写。

## 💡 重要洞见
**峰终定律的实际意义**：学员整堂课听下来，最后记住的只有两件事——最精彩的一个点（顶峰）和最后几分钟（结尾）。所以结尾的设计和顶峰的设计同样重要，甚至更重要。把最好的截屏点放在结尾附近，会大幅提升「有收获」的感觉。
