# 第三章：如何生产一门知识小课

> **本章核心**：知识小课是新讲师最容易切入的类型，1周可以做出一门，熟练后3天即可。关键是主题聚焦、交付清晰、截屏点设计到位。

---

## 🎯 核心主张
知识小课不需要话题特别吸睛或讲师背景有噱头——只要内容做扎实，就有反复观看的价值，能形成长尾效应。

---

## 📌 关键框架

### 三种类型的知识小课

| 类型 | 定位 | 课程结构 | 学员期待 |
|------|------|---------|---------|
| 新知新见类 | 拓宽视野，了解新概念 | 总（引兴趣）→分（原理+案例）→总（截屏点）| 了解新事物，不必马上应用 |
| 工作方法类 | 解决一个具体业务问题 | 总（激发欲望）→分（步骤+成果标准）→总（加深记忆）| 马上用在工作里 |
| 工具实操类 | 跟着操作，学完即可用 | 总（展现成果预期）→分（操作步骤）→总（路径总结）| 做出实体成果 |

**重要限制**：工作方法类课程讲授的方法**必须是本人实践过3次以上**的，不能道听途说。

### 截屏点「1234」设计标准（每门课至少要有，最关键的一步）

| 形态 | 用途 | 做法 |
|------|------|------|
| 1清单 | 步骤/自检/注意事项 | 列出所有要检查的项目，方便对照 |
| 2对比 | 两种方案/新旧对比 | 用表格或左右对比展示差异 |
| 3支柱 | 三要素关系 | 维恩图、三角图、三列卡片 |
| 4象限 | 两维度判断决策 | 坐标轴，用两个关键变量构建象限 |

检验标准：截图发给没看过这门课的人，10秒内能看懂吗？

## 🔧 可落地的交付物

### 快速判断课程类型（做课前先填）

```
我想讲的内容是：_______________

□ 我要告诉学员一个他们可能不知道的新概念或新现象
  → 新知新见类，重点讲清楚Why（为什么重要）和What（是什么）

□ 我要帮学员解决一个具体的工作问题，学完就能用
  → 工作方法类，重点把操作步骤讲清楚，每步有成果标准

□ 我要教学员用一个工具/软件，跟着操作能做出东西
  → 工具实操类，重点展示最终成果，逐步演示操作
```

### 知识小课完整自检清单

**教学目标**：
- [ ] 目标学员是哪一类？（没概念/有了解/有经验，只选一类）
- [ ] 学习成果定义清晰吗？（学完能做什么，而不是「学到了知识」）
- [ ] 课程时长和内容量匹配吗？（30分钟-2小时，不宜超过）

**课程结构**：
- [ ] 开头前60秒，有没有让目标学员感受到「这跟我的工作有关」？
- [ ] 课程里有没有至少一个清晰的截屏点（1234之一）？
- [ ] 结尾是否强化了应用场景，而不是简单重复目录？
- [ ] 如果是工作方法类，这个方法我实践过3次以上吗？

**内容质量**：
- [ ] 核心方法论挖到了「学员可以按方法实践」的颗粒度？
- [ ] 有没有至少一个真实具体的案例（不是泛泛的「很多公司都会」）？
- [ ] 交付物（工具/清单/模板）学员拿到手就能用吗？

### 概念生命周期判断工具（适用于新知新见类）

做课前先判断这个概念处于哪个阶段：

| 阶段 | 特征 | 讲什么 |
|------|------|--------|
| 引入期 | 大多数人没听说过 | 背后的原理，让人知道它的存在 |
| 推广期 | 有人开始应用并取得成果 | 应用范畴和成功案例 |
| 普及期 | 大众都知道了 | 细分场景的具体应用 |
| 创新期 | 已成为标配 | 推陈出新、前沿创新应用 |

> 在「普及期」还在讲「这个概念是什么」就没有竞争力了，要切到更细分的应用场景。

---

## 💡 重要洞见
知识小课做多了，自然会发现它们之间的联系——有些知识小课可以组合成体系课程的雏形。所以知识小课不只是独立产品，也是探索「这个方向值不值得做大」的低成本方式。
