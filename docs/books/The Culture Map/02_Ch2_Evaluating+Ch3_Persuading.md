# Ch2：The Many Faces of Polite（负面反馈的直白程度）

## 🎯 核心主张

**"说话直白"是一个维度，"给负面反馈直白"是另一个维度，两者不一定一致。**美国人平时直白，给负面反馈却包成三明治；法国人平时含蓄，给负面反馈却毫不留情。

## 📌 关键框架/方法论

### 4 象限：把 Communicating × Evaluating 两个维度叉乘

| | 低语境（说话直白） | 高语境（说话含蓄） |
|---|---|---|
| **直接给负面反馈** | A 象限：荷兰、德国、丹麦 | B 象限：俄罗斯、以色列、法国 |
| **委婉给负面反馈** | C 象限：美国、英国、加拿大、澳洲 | D 象限：日本、中国、印尼、泰国、墨西哥 |

**反直觉的点**：
- 美国人平时说话最直白，但给负面反馈时反而是世界上最委婉的之一
- 法国/俄罗斯人平时说话含蓄，但骂起人来非常凶
- 以色列是高语境 + 极直接（在 B 象限里最极端）

### Upgraders vs Downgraders（升级词 vs 降级词）

判断一个文化负面反馈的直白程度，看他们用什么词包装。

| Upgraders（让批评更重） | Downgraders（让批评更轻） |
|---|---|
| absolutely | kind of |
| totally | sort of |
| strongly | a little / a bit |
| completely | maybe / possibly |
| | slightly |

**英国人是 downgraders 之王**。

### Anglo-Dutch 翻译指南（经典）

| 英国人说 | 英国人意思 | 荷兰人/直白文化听到的 |
|---|---|---|
| "I hear what you say" | 我不同意，不想再讨论 | 他听到我了 |
| "With the greatest respect..." | 你是个白痴 | 他在认真听我 |
| "That's not bad" | 很好 | 不好 |
| "Quite good" | 一点点好而已（有点失望） | 不错 |
| "Perhaps you would like to think about..." | **马上改，否则有麻烦** | 我考虑一下 |
| "Very interesting" | 真烂 | 他们觉得有趣 |
| "I'll bear it in mind" | 我已经忘了 | 他们会照做 |
| "I'm sure it's my fault" | 这都是你的错 | 他们觉得是自己的错 |
| "You must come for dinner" | 客套话 | 我马上要被邀请了 |

## 🔧 可落地的交付物

### 工具 1：四象限对应的反馈策略

**应对 A 象限（荷兰/德国，低语境 + 直接）**
- 别为他们的直白生气——这是尊重你的表现
- **不要**模仿他们的风格（韩国经理 Kwang Young-Su 的反面教材：以为要"以毒攻毒"，结果被孤立）
- 一次只升级一个档位，慢慢调

**应对 B 象限（俄罗斯/以色列，高语境 + 直接）**
- 老板对下属可以严厉，下属对老板要外交
- 对陌生人会更凶（俄罗斯文化遗产：陌生人 = 潜在的敌人）
- 知道这是文化特征，不要往个人攻击上联想

**应对 C 象限（美国/英国，低语境 + 委婉）**
- 三明治原则：先说优点，再说缺点
- **正面反馈必须具体**（不能空洞地说 great/excellent）
- 反馈量要平衡：今天看到好事立刻当面表扬，明天批评才能被听进去
- 用文化框架解释自己："In France, we don't usually voice positive feedback"
- 给个翻译表："When I say okay, you should hear very good. When I say good, you should hear excellent."

**应对 D 象限（亚洲/墨西哥，高语境 + 委婉）**
- **绝对不要在团队面前给个人负面反馈**（Charlie Hammer 的墨西哥下属因此辞职）
- 用 Aini 的"模糊三策略"

### 工具 2：Aini 的 Blur the Message 三策略（亚洲场景）

**策略 1：分阶段，慢慢传递**
- 不要一次说完
- 在不同场合零散地暗示需要改进的地方
- 给对方留消化时间

**策略 2：用饭局/酒局包装**
- 不要在办公室正式叫进来谈
- 约出去吃饭，氛围放松了再说
- 第二天回办公室像没发生过一样
- 信息传到了，但没破坏面子

**策略 3：Say the good, leave out the bad**
- 4 份文件，前两份好后两份差 → 只夸前两份，详细解释为什么好
- 对方自然懂"言外之意"
- 你没说一句批评，但对方收到了完整信息

### 工具 3：自检——你的反馈风格是什么

回答这两个问题，找到自己的象限：

1. 当我说"this is great"，我有多频繁说这个词？
   - 几乎不说 → 直接型（B 象限或 A 象限）
   - 一周几次 → 委婉型（C 或 D 象限）

2. 当我对工作不满意，我会：
   - 用 "absolutely terrible"、"this is unacceptable" → 直接型
   - 用 "maybe we could think about..." → 委婉型

### 工具 4：跨文化批评的安全 default

不知道对方在哪个象限时，用这个保守套路：

1. **私下**说，不要在团队前
2. **以问句开头**："I'm curious how you're thinking about X..."
3. **明确说出文化背景**："我在国内习惯比较直白，如果有冒犯请告诉我"
4. **写下来 follow-up 邮件**，让对方有消化和回看的时间
5. **观察对方的反馈风格**反推他的象限，下次调整

## 💡 重要洞见与案例

### 案例 1：英航机长 Eric Moody（1982 火山灰事件）

四个引擎全停了。机长广播："Good evening again, ladies and gentlemen. We have a small problem in that all four engines have failed. We're doing our utmost to get them going and I trust you're not in too much distress."

经典英式 understatement。**飞机最后滑翔着落，零伤亡。**

### 案例 2：Marcus Klopfer 差点丢工作

德国经理 Klopfer 的英国老板说 "I suggest that you think about doing it differently."
Klopfer 真的"think about it"了，然后决定**不**改。
两周后被老板叫去办公室"chew me out for insubordination"。

**教训**：英国人的 "perhaps you would like to think about" = 德国人的 "you must change this now"。

### 案例 3：法国小学的红叉

Erin 的儿子 Ethan 在法国小学。一份听写错了 8 个词，老师 Madame Durand 用红笔打满 X，写："8 errors. Skills not acquired. Apply yourself!"

这个老师是这个学校**最温柔**的。

### 案例 4：Sabine Dulac 在芝加哥的灾难

会议是个灾难，所有美国同事先夸"booth 摆得好"再说问题。Sabine 直接说"keynote uninspiring, food inedible, breakout boring"。美国同事像见鬼。

**教训**：在 C 象限文化里，**先说好处再说问题不是虚伪，是协议**。

### 案例 5：Aini 教 Erin 怎么解雇 Budi

Erin 要从节目里砍掉印尼资深顾问 Budi 的两个 session，怕伤害关系。
Aini 教她：
1. 第一封邮件：泛泛说"基于反馈我要重新设计节目"
2. 电话里：暗示有些 session 会调整
3. 午餐时（巴黎香榭丽舍餐厅）：只夸 Budi 第一个 session 好，说要"build on"它
4. Budi 回："Sounds great! 短一点 impact 大反而好"

**完全没说一句批评，目的全达到，关系没破。**

# Ch3：Why Versus How（说服的两种逻辑）

## 🎯 核心主张

**说服别人，对方关心的可能不是"做什么"，而是"为什么这么做"。**美国/英国人喜欢直接看结论；德国/法国/南美/亚洲人需要先理解"why"才能接受"how"。

## 📌 关键框架/方法论

### Persuading 维度坐标

```
英美澳加新 < 北欧/荷兰 < 拉美/德国 < 法/俄/比
（applications-first，先讲案例）  →  （principles-first，先讲原理）
```

亚洲文化不在这个维度上——他们用的是另一套：**holistic（整体）vs specific（局部）**。

### 两种推理方式

| | Applications-first（归纳） | Principles-first（演绎） |
|---|---|---|
| 出发点 | 具体案例、观察 | 一般原理 |
| 路径 | 观察 → 提炼规律 → 结论 | 原理 → 应用 → 结论 |
| 数学课 | 先教公式怎么用，再讲原理 | 先证明原理，再推导公式 |
| 语言课 | 沉浸式直接说，从语境推语法 | 先学语法，再开口 |
| 法律体系 | Common Law（判例法，英美） | Civil Law（成文法，欧陆） |
| 商业沟通 | "Here's my recommendation, here's the data" | "Here's the framework, here's the methodology, here's the conclusion" |
| 出处哲学 | Aristotle, Bacon | Descartes, Hegel |

### 第三种逻辑：东亚的 Holistic Thinking

Nisbett & Masuda 的研究：
1. **水族箱实验**：美国人先描述前景大鱼；日本人先描述背景植物和小鱼之间的关系
2. **拍人像实验**：美国人拍脸特写；日本人把人放在环境里拍小
3. **看默剧实验**：美国人 30% 的描述围绕主角；台湾人围绕主角的比例低得多

### 中西思维顺序对比

| | 中国 | 西方 |
|---|---|---|
| 地址 | 国 > 省 > 市 > 区 > 街 > 号 | 号 > 街 > 区 > 市 > 省 > 国 |
| 姓名 | 姓 + 名 | 名 + 姓 |
| 日期 | 年/月/日 | 日/月/年 |
| 思维 | macro → micro | micro → macro |

中国人的思路：**先看大局，再看局部**。
西方人（尤其美国人）的思路：**先看局部，再推大局**。

## 🔧 可落地的交付物

### 工具 1：根据 audience 调整说服顺序

**演讲给美国人/英国人（applications-first）**
- 开头一句话讲结论："I recommend we do X."
- 接下来举 2-3 个具体案例
- 最后简短说一下方法论
- iPhone 屏幕装得下的邮件才会被读

**演讲给德国人/法国人（principles-first）**
- 开头 5 分钟讲清楚方法论："这是我研究的范围、样本、方法"
- 再展示数据
- 最后才说结论和建议
- 期待并欢迎质疑（这是流程的一部分，不是不尊重）

**演讲给日本人/韩国人/中国人（holistic）**
- 开头先讲**整体图景**：这件事属于什么大背景、和其他事的关系
- 解释各方利益相关者会如何受影响
- 给的指令越细越好，但每个指令要嵌在整体里
- 不要孤立讲一个人的目标和奖金——团队会自发协商重新分配

### 工具 2：写邮件的"屏幕测试"

Stéphane Baron 的法国邮件结构（典型 principles-first，4 段铺垫）→ 英国同事直接归档不读。

新规则：**iPhone 屏幕一屏装不下的邮件，不会被英美人读。**

写给英美人的邮件结构：
1. 第一行：你想要他做什么
2. 接下来 2-3 个 bullets：为什么这样做
3. 结尾：deadline 和 next step

### 工具 3：跨文化团队的 4 条具体建议

来自 Erin 给跨文化 audience 演讲的实战：
- 让大家都读这一章，统一术语
- 在团队里找 1-2 个"文化桥梁"（有跨文化经验的）负责翻译
- 主动适应对方的行为
- 耐心 + 灵活——跨文化效率本来就慢

### 工具 4：组建多文化团队的两个原则（重要）

来自本章末尾，但非常实战：

**原则 1：少数人跨文化，多数人做本地化**
- 不要让所有人都跨文化合作
- 每个国家选 1-2 个**最国际化**的人来做跨界
- 其他人留在本地，按本地方式工作
- 这样既有创新，又不被磨擦消耗

**原则 2：根据目标决定是否混编**
- 目标是创新/创意 → 多元文化（差异是养分）
- 目标是速度/效率 → 单一文化（差异是摩擦）
- **"Sometimes, it is simply better to leave Rome to the Romans."**

### 工具 5：演讲给混合 audience 时的"交替节奏"

来自 Jens Hupert 的实战：

```
讲实例（吸引 applications-first 听众）
↓
有人问理论问题
↓
认真回答（不要敷衍，否则失去 principles-first 听众尊重）
↓
回答完立刻补一个新例子（把 applications-first 听众拉回来）
↓
循环
```

## 💡 重要洞见与案例

### 案例 1：Kara Williams 的德国失败

美国工程师，到德国汽车公司，做了 3 个月调研。直接讲结论。德国总监立刻打断："How did you get these conclusions? What methodology?"
Kara 觉得这是不尊重她的专业，防御性反应，演讲彻底毁了。

**教训**：在德国，质疑方法论 ≠ 不尊重你，而是流程必经环节。

### 案例 2：Stéphane Baron 的英国邮件没人读

法国精英写邮件用 4 段：引入 → 论证 → 反驳 → 结论。
英国同事看第一段没看到结论，归到"以后再说"文件夹，从不打开。

### 案例 3：Lilly Li 的 holistic 回答

Erin 问："How to manage confrontation on the team?"
Lilly Li 先讲 trust（不是 confrontation），讲她在匈牙利的经历（案例里没有匈牙利人），讲了 5 分钟，最后才回到 confrontation。

Erin 一开始以为她跑题，后来发现：**这就是 holistic 思考**——她在搭建整个"为什么 confrontation 在我团队里成立"的大背景。

### 案例 4：巴西 Jorge Da Silva 怎么搞定美国总部

Da Silva 的新安全流程，发了 10 个详细 PPT 给美国办公室，越解释越没人听。
他朋友 Jake 说："Have you tried showing them an example of what could happen if it's well implemented?"
Da Silva 邀请美国决策者到巴西工厂亲眼看 2 天 → 回去就批了。

**教训**：对 applications-first 文化，**给案例比给原理有效 10 倍**。

### 案例 5：Jacek Malecki 在日本的失败

波兰经理在日本逐个谈话，定个人目标和奖金。日本团队回去自己重新分配了任务，按整体协同来。

**教训**：在 holistic 文化里，**个人目标 + 个人奖金 + 切片任务**这套打法不灵——他们要先看清整体怎么咬合。
