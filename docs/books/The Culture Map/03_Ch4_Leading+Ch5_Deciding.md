# Ch4：How Much Respect Do You Want?（领导力的等级感）

## 🎯 核心主张

**当老板的"权威感"从哪里来？**在平等型文化里来自"和大家一样"；在等级型文化里来自"明显和大家不一样"。同样的动作（骑车上班、坐 open space），在不同文化里发出完全相反的信号。

## 📌 关键框架/方法论

### Leading 维度的坐标（基于 Hofstede 的 Power Distance）

```
丹麦/瑞典/荷兰 < 美国/澳洲/英国 < 德国 < 法/意/西 < 拉美 < 印度 < 中国/韩国/日本/沙特/俄
（egalitarian 平等）  →  （hierarchical 等级）
```

### 决定一个文化"等级感"强弱的三个历史因素

1. **罗马帝国遗产**（南欧）：拉丁文化（西、意、法部分）强等级——元老穿白袍紫边，皇帝穿紫袍，阶级一眼可见
2. **维京文化**（北欧）：丹麦、瑞典、挪威——维京人定居冰岛时就建立早期民主，"开会大家投票"
3. **新教 vs 天主教**：新教国家（直接和上帝对话）更平等；天主教国家（要通过神父、主教、教皇）更等级
4. **儒家文化**（东亚）：孔子的 **wu lun 五伦**——君对臣（仁/忠）、父对子（保护/孝）、夫对妻（义/顺）、兄对弟（爱/敬）、友对友（信）。"老板对员工" 是隐含的第六伦。

### 平等型 vs 等级型领导对比（书中 Figure 4.3 提炼）

| | 平等型文化（北欧/荷兰） | 等级型文化（亚洲/拉美/俄罗斯） |
|---|---|---|
| 老板形象 | facilitator among equals | 像"父亲"或"君主" |
| 称呼 | 直接叫名字（Ulrich, Karl） | 用职位+姓（Mr. President, 王总） |
| 办公场所 | open space，没门 | corner office，独立办公室 |
| 着装 | 和员工一样随便 | 明显升级 |
| 跳级沟通 | 可以直接 email 大老板 | 必须经过直接上司 |
| 老板回答问题 | "你自己想"（瑞典 7% 觉得老板要有答案） | "我告诉你怎么做"（意大利 55% 觉得老板要有答案） |
| 决策速度 | 慢（要听所有人意见） | 快（老板拍板） |

## 🔧 可落地的交付物

### 工具 1：跨文化"符号检查清单"

你的日常行为在不同文化里发出的信号完全不同。出海前自检：

| 行为 | 在平等型文化 | 在等级型文化 |
|---|---|---|
| 骑车上班 | "他是我们一员"（加分） | "丢人，老板像穷人"（团队替你尴尬，扣分）|
| 让员工叫名字 | 正常（"Ulrich"） | 别扭，建议用 "Mr. Mike" 混合称呼 |
| 没有自己的办公室 | 平等（加分） | "他公司里不重要，连办公室都没有"（扣分） |
| 在电梯里被员工邀请午餐 | 正常 | 越级，需要先报上司 |
| 给下属的下属发邮件 | 直接沟通 | 必须 cc 中层 |

### 工具 2：在等级型文化里当老板的 5 个动作（Confucian 老板的责任）

来自 Steve Henning 在北京 6 年的总结：

1. **承担"父辈式"责任**：不只是要求服从，更要保护、指导、关心团队的利益
2. **明确给指令**：等级型文化下属在等你拍板，不要逼他们"自己想"
3. **会议前几天给提纲**：告诉团队你会问什么问题，他们才有机会准备
4. **会议中点名邀请发言**：不要期待有人主动跳出来——他们在等你叫名
5. **私下问意见**：让团队**没有你参加**地讨论，再把结果汇报给你

### 工具 3：在平等型文化里当老板的 3 个动作

来自 Carlos Gomez 在荷兰的总结：

1. **MBO（目标管理）替代日常控制**：让员工自己提目标→你协商→定下来，你只在月度 review 时介入
2. **不要 cc 老板**：在荷兰/北欧，cc 上司 = 我不信任你，想给你穿小鞋
3. **着装、用语全部 downgrade**：脱掉西装、用名字、轮值主持会议

### 工具 4：Level-Hopping（跳级沟通）的安全策略

不知道对方文化在哪？**默认走等级型规则**——更安全，不容易出错。

| 你是 | 行动 |
|---|---|
| 想跨级 email 大老板 | 先和直接上司商量，得到允许 |
| 想 email 下属的下属 | cc 中层 |
| 写 email 给亚洲/拉美等级文化的人 | 用姓 + 职位，除非对方先用名字签名 |
| 写给斯堪的纳维亚/荷兰/澳洲 | 直接用 first name |
| 不确定 | 用姓 + Mr/Ms 开头，先试探 |

### 工具 5：自检——你的等级感是从哪里来的？

判断你天然偏好哪一端：

- 在会议里有人 push back 你的决定，你的第一反应是"好，我听听"还是"他怎么敢"？
- 你的下属主动给大老板发了 critique，你的反应是"挺好"还是"完了我要被骂"？
- 员工开会主动发言但意见不成熟，你觉得是"敢说话好"还是"乱说话"？

如果你三个都偏后者，你天然是等级型——出海到平等文化要做调整。

## 💡 重要洞见与案例

### 案例 1：Ulrich Jepsen 的"president"困境

丹麦经理，习惯没办公室、open space、被叫名字。被派去管俄罗斯团队 4 个月后给 Erin 发邮件抱怨：
- 他们叫我 Mr. President
- 他们什么决定都让我拍
- 他们不主动
- 他们把我当国王

**同时俄罗斯团队也在抱怨他**：
- 他是个软弱无效的领导
- 他不会管理
- 他放弃了顶层 corner office，让公司看不起我们团队
- 他无能

**教训**：在等级型文化里，"和大家一样"= "我没分量"。

### 案例 2：Carlos Gomez 在 Heineken（墨西哥 → 荷兰）

墨西哥经理到阿姆斯特丹管荷兰团队。
- 行政助理 Karl 在电梯里直接和大老板的老板约了午餐，没告诉 Carlos
- 直接下属 email 给 CEO 提意见（公司 64,000 人），完全没 cc Carlos
- Carlos："I felt my smile spread and freeze across my face"

**教训**：荷兰人的"跳级"在墨西哥 = 当面打老板的脸。

### 案例 3：Steve Henning 的自行车

澳洲人爱骑车上班，在墨尔本团队觉得老板平易近人。
搬到北京后继续骑车 → 中国团队**集体觉得丢脸**，认为老板没排面，他们自己也没排面。
最后他换公共交通。

### 案例 4：Anne-Hélène 用 Pam 的电脑

法国人在美国小公司工作，电脑坏了，美国同事说："用 Pam 的吧，她不在，open door policy。"Pam 是公司总裁。
她还记得开 Pam 玻璃门、走到办公桌前、按下键盘那一刻"感觉总裁的气场还在房间里"。

### 案例 5：Sarah Peterson + Rishi Rangan 的越级邮件灾难

加拿大经理 Sarah 直接 email 印度团队成员（Rishi 的下属），下属们害怕得不敢回，三次也不回。
Sarah 觉得"印度团队不靠谱"。
Rishi 觉得"Sarah 故意绕开我"。

最后**花大钱飞 9 个人去班加罗尔 3 天面对面谈**才解决。

**教训**：跨文化合作早期不商量沟通协议，后期付的代价 10 倍。

# Ch5：Big D or Little d（决策的两种模式）

## 🎯 核心主张

**"决策"在不同文化里意思完全不同。**美国老板的 "decision" 是个小 d——可以随时改、随时根据新信息调整；德国/日本的 "decision" 是大 D——一旦做了就是承诺，按计划推进。两种模式都行，混在一起就是灾难。

## 📌 关键框架/方法论

### Deciding 维度的坐标

```
日本/瑞典/荷兰/德国/法国 < 巴西/英国/澳洲 < 印度/中国/沙特 < 美国/俄罗斯
（consensual 共识）  →  （top-down 自上而下）
```

### 反直觉：Leading 和 Deciding 不一定一致

大多数文化两个维度位置接近（平等 + 共识 / 等级 + 自上而下），但有 3 个重要例外：

| 国家 | Leading | Deciding | 矛盾点 |
|---|---|---|---|
| 美国 | 平等（偏中） | 自上而下 | 平等型 + 快速拍板 |
| 德国 | 等级（偏中） | 共识 | 等级型 + 慢慢讨论 |
| 日本 | 极等级 | 极共识 | 等级 + 共识（ringi 制度）|

这就是为什么美国 + 德国合作互相抱怨"对方好等级"，但抱怨的是**不同维度**。

### 两种决策模式的时间线对比

**共识型（德国/日本/瑞典）**
```
讨论 ━━━━━━━━━━━━━━━━━━━━━━━━━━━ 决策点（Big D）━━━ 实施
（很长，所有人参与）              （定了就不改）   （快）
```

**自上而下型（美国/中国/俄罗斯）**
```
决策点（little d）━━━ 不断调整、补充、修改 ━━━ 最终实施
（很快，老板拍板）   （来回反复）          （慢）
```

### 日本的 Ringi 系统（最极端的共识 + 等级共存）

`ringisho`（稟議書）= 提案文件
`nemawashi`（根回し）= 字面意思"包根"——移植树前先把根包好，引申为正式开会前先私下打招呼

流程：
1. 中层提案，写 `ringisho`
2. 每个相关人读、改、盖章
3. 全部盖完往上一层送
4. 上一层再讨论一遍达成共识
5. 一直到最高层
6. 决策定了 = **不可改**，但实施飞快

### 美国和德国对"决策"的根本不同理解

| | 美国理解 | 德国理解 |
|---|---|---|
| "We've made a decision" | "我们达成一致继续讨论了" | "我们做出了一个承诺" |
| 改变决策 | 正常，根据新信息随时改 | 不可接受，等于背叛 |
| 决策时间 | 越快越好 | 越充分越好 |
| 老板的角色 | 拍板的人 | 收集共识的人 |

## 🔧 可落地的交付物

### 工具 1：跨文化决策协议的 6 个 must-discuss

任何跨文化项目启动时，**必须**先把这 6 个问题摆桌面：

1. 这是 Big D 还是 little d？做了之后能不能改？
2. 决策方式：投票？老板拍？共识？多数同意（百分比阈值）？
3. 谁有发言权？谁有否决权？
4. 有没有 deadline？过了 deadline 还能讨论吗？
5. 决策做完后改的成本：流程上需要谁同意？
6. 团队成员是否都接受这套规则？

### 工具 2：面对共识型团队的 5 个动作

来自给和瑞典/德国/日本人合作的人的建议：

1. **预期时间是平时的 2-3 倍**
2. **保持耐心 + 频繁出现**，否则会被遗忘
3. **培养非正式的"线人"**：在共识形成过程里你能知道方向走到哪了
4. **不要催**：催 = 不尊重流程
5. **质量优先**：决策定下来后改不动，所以前期做对最重要

### 工具 3：面对 top-down 团队的 5 个动作

来自给和美国/中国/俄罗斯老板合作的人的建议：

1. **预期决策可能在你不在场时就做了**
2. **接受老板拍板**，哪怕你的意见被否
3. **当你是老板时，要快**：listen 也要快，否则被认为软弱无效
4. **不知道怎么办时投票**：投了就跟着走
5. **保持灵活**：决策不是石头，是橡皮泥

### 工具 4：日本场景的"根回"协议

如果你要去日本/韩国争取一个决策：

- **不要把宝压在会议上**——会议只是盖章
- **提前 3-4 周开始 nemawashi**：私下一对一聊关键人物
- **从中层往上铺**，不要直接找 CEO
- **每个人都要听到你的提案**，让他们觉得是"自己人"
- **会议上让最高层"宣布"**，但其实结果几天前就定了

### 工具 5：D/d 翻译表

帮你的团队建立共同语言：

| 美国老板说 | 等于德国老板说 |
|---|---|
| "Let's go with this." | "I think we should explore this option further" |
| "Decision made!" | "We've reached a tentative agreement" |
| "I changed my mind based on new data." | "We need to formally re-open the decision" |

新协议示例（Erin 帮助 美国/德国 merger 团队建立的）：

> 美国经理："Great! Decision made! ——Decision with a small d, that is! 我们还得跟国内同事确认，先别动工。"
>
> 德国经理："So have we agreed on a decision? small d or big D?"

## 💡 重要洞见与案例

### 案例 1：纽约-慕尼黑并购的双向抱怨

**美国 Larry**："这帮德国人太等级了！我和一个慕尼黑分析师吃个午饭，居然被他老板的老板教训，说我跨级了。"

**德国 Matthias**："这帮美国人才等级！老板说 'march left' 全员立刻向左转，没人质疑。我作为德国人 push back 一下，几个月后就要丢工作了。"

**真相**：他们各对，但说的是不同维度——美国是 top-down decision，德国是 hierarchical leading。

### 案例 2：Erin 第一次和瑞典老板共事

瑞典老板 Per 自我介绍："我是 consensus-building 风格的，请耐心一些。"
Erin 以为这意味着"老板会听大家意见"，觉得很棒。
两周后开始收到决策邮件——所有人在 email 里来来回回讨论年会主题。Erin 没意见、删了。
最后 Per 单独给她发："Hi Erin, 没听到你的意见，你怎么想？"
Erin 想说："我没意见，你拍板就行。"
后来明白：在瑞典，没参与共识 = 不负责任。

### 案例 3：Jack Sheldon 输给"空气"

英国 Astellas 经理飞东京开会，做了 3 周准备，要说服日本团队继续做某产品测试。
进会场，日本同事很客气；一个日本经理开场陈述了"停止测试"的论点；Jack 讲完幻灯片，气氛礼貌但**完全不动**。
后来 Susumi Mori 告诉他："决策其实早在 nemawashi 阶段就定了，会议只是盖章。"

**教训**：在日本，**会议是结果，不是过程**。要影响决策，必须在会议**之前** 3-4 周开始铺垫。
