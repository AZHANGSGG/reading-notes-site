# 第二部分：构建信任的框架（第8-14章）

> **本部分核心**：把「信任」这件感性的事情结构化——不只是知道要建立信任，而是知道**怎么做**、**从哪里入手**、**卡在哪一步**。

---

## 第8章 信任等式

### 🎯 核心主张
信任可以被分解为四个要素，可以被评估和改善——每次见完客户都可以打分，找到提升的方向。

### 📌 关键框架

```
T = (C + R + I) / S

C = 可信度 Credibility  （内容和举止上的专业）
R = 可靠度 Reliability  （言出必行，始终如一）
I = 亲近感 Intimacy     （能否谈论棘手问题）
S = 自我导向 Self-orientation  （越低越好）
```

**新老客户对比**：

| 场景 | C | R | I | S | 信任值 |
|------|---|---|---|---|--------|
| 新客户 | 5 | 3 | 2 | 8 | 1.25 |
| 老客户 | 7 | 8 | 5 | 4 | 5.00 |

信任值相差4倍——这正是开发新客户的成本是维护老客户4-7倍的根本原因。

### 🔧 可落地的交付物
**会后信任等式自评表**（每次见完重要客户后用，1-10分）：

| 维度 | 自评分 | 本次具体表现 | 下次改进点 |
|------|--------|-------------|-----------|
| C 可信度 | /10 | | |
| R 可靠度 | /10 | | |
| I 亲近感 | /10 | | |
| S 自我导向（越低越好）| /10 | | |
| **信任值 T = (C+R+I)/S** | | | |

**各维度立刻可做的事**：
- C提升：见面前做深度调研，带着有价值的洞察去，不只是来听
- R提升：在小事上做出具体承诺（「我今天下班前发给你」），不露声色地兑现
- I提升：找一件真实但稍微敏感的事说出来，先承担责任再提问题
- S降低：开会时，先问「你今天最想讨论的是什么？」再说自己的议程

### 💡 重要洞见
**S（自我导向）是最关键的分母**——客户能敏锐地感知你的注意力在哪里。总把他们的故事与自己联系、急于帮他们把话说完、急于给出答案，都是S过高的信号。

## 第9章 信任关系的建立（五阶段）

### 🎯 核心主张
建立信任有一套可以被学习的五阶段流程——每个阶段有不同的目的、语言风格和技能要求。

### 📌 关键框架

| 阶段 | 顾问目标 | 客户感受 | 标志性语言 |
|------|---------|---------|-----------|
| 1. 委托 | 让客户意识到问题值得与你讨论 | 「也许这个人能帮上我」 | 表达关心和兴趣 |
| 2. 倾听 | 让客户相信你理解他们的问题 | 「他真的在听我说话」 | 确认、追问、共情 |
| 3. 界定问题 | 帮助客户从新角度认清问题 | 「原来问题是这样的」 | 直接、坦率、有视角 |
| 4. 构想未来 | 对齐目标和愿景 | 「我看到了不同的可能」 | 描述可能性的语言 |
| 5. 立下承诺 | 确保客户理解并愿意行动 | 「我知道要做什么了」 | 共同合作的语言 |

**最常见的错误**：从「委托」直接跳到「承诺」，跳过中间三步——急着给答案。

### 🔧 可落地的交付物
**卡点诊断工具**——每次客户关系遇到阻力，问自己：

```
客户不愿意开口？        → 卡在「委托」→ 我带来了什么值得他说的东西？
客户觉得你不理解他？     → 卡在「倾听」→ 我在听我想听的，还是他说的？
客户同意但不行动？       → 卡在「界定」或「构想」→ 我们对问题的理解一致吗？对目标的理解一致吗？
双方都同意但事情没推进？  → 卡在「承诺」→ 回去检查前面几步是否真的做扎实了
```

> 当客户对承诺存疑时，**往回退一步甚至两步**，到上游去找原因。

## 第10章 赢得客户委托

### 🎯 核心主张
客户不会仅仅因为你愿意倾听就开口——你必须先用行动触动他们，让他们相信「这个人值得我说」。

### 📌 关键框架
- **新客户**：在见面前做深度调研，带着真正有价值的洞察去，不是「我愿意听你说」，而是「我带来了你感兴趣的东西」
- **老客户**：在无特殊意义的某天主动联系，持续关注他的行业和竞争对手

### 🔧 可落地的交付物
**见面前准备清单**（每次拜访客户前填写）：
- [ ] 我了解他最近面临的最大挑战是什么？
- [ ] 我有没有一个他的行业里发生的事情，是他可能还不知道但会感兴趣的？
- [ ] 我能用一句话说清楚「这次见面对他有什么价值」吗？
- [ ] 我是带着问题来的，还是带着答案来的？（应该是问题）

**主动联系老客户话术**：
> 「最近看到XX行业发生了YY，我想到了你之前说的ZZ，不知道对你的判断有没有什么影响，想顺便聊几分钟。」——不带商业目的，只是真诚地想到了他。

## 第11章 倾听的艺术

### 🎯 核心主张
倾听不是礼貌，是赢得权利——在没有有效倾听的前提下直接给建议，会被客户读成傲慢。

### 📌 关键框架
**两种常见错误**：
- **过分理性的倾听**：只听内容，不听情绪
- **过分被动的倾听**：没有积极回应，只是「嗯哈嗯哈」

**优秀倾听者的行为清单**：
- ✅ 持续追问以澄清理解
- ✅ 留意情绪表达，而非只听内容
- ✅ 让对方按自己方式把故事讲完
- ✅ 在告诉你他们想法前，先征求你的想法
- ❌ 打断别人
- ❌ 过快回应（还没消化就开口）
- ❌ 把自己的故事与客户的故事相比
- ❌ 开会时接电话

### 🔧 可落地的交付物
**会议开场话术**（传递「这次会议是为你服务」的信号）：

> 「我准备了几个想聊的点，包括A和B。但在开始之前，你今天最想讨论什么？」

**倾听自检**（会后回顾）：
- 我有没有在他说话时，脑子里已经在想自己的答案？
- 我有没有注意到他的语气变化？（激动？迟疑？）
- 我有没有允许沉默存在，还是急着填满空白？

> GE总裁韦尔奇评价他的律师：「他的倾听技巧比任何人都要出色。」——注意，是倾听技巧，不是法律技巧。

## 第12章 界定问题

### 🎯 核心主张
准确描述问题，本身就等于解决了问题的一半——界定问题是五阶段中最困难但回报最丰厚的一步。

### 📌 关键框架
**两种界定方式**：
- **理性界定**（顾问的舒适区）：把问题剖解到关键变量，用模型和框架结构化
- **感性界定**（更难但更关键）：把「房间里的大象」说出来，需要勇气

### 🔧 可落地的交付物
**「直言不讳」铺垫话术模板**（下次想说但不敢说时用）：
```
「也许只有我这样觉得，但是……」
「我可能完全想错了，但是……」
「这件事不知当讲不当讲，但是……」
「我希望有办法，但我实在不知道如何处理这个困扰……」
```
共同点：**先承担责任，再提问题**——让对方感受到你不是在攻击，是在冒险开口。

**界定 vs 责备的检验**（马克案例）：
- ❌ 「客户待我实在不公」（责备）
- ✅ 「公司事务变得繁忙，他没能抽出原本想要的时间，这与他的初衷不符，也使我面临交付风险——**我们两人对这个局面都不满意**。」（界定）

检验标准：你的表述里有没有「我们」？有没有对双方处境的理解？

## 第13章 构想另外一种情景

### 🎯 核心主张
在界定问题之后，大多数人急于进入解决方案——但如果不先对齐目标和愿景，行动再多也是白费。

### 📌 关键框架
**构想阶段必须回答的三个问题**：
1. 我们的真正目标是什么？
2. 达成目标意味着什么（对他个人意味着什么）？
3. 我们怎样判断已经达成目标？

**语言信号切换**：
- 界定阶段用：「问题是……」
- 构想阶段用：「如果我们……将会……」

### 🔧 可落地的交付物
**每次大型提案或复杂项目前，先问客户这三个问题**（不要跳过）：
> 「在我们讨论具体方案之前，我想先确认我们对目标的理解是一致的。在你看来，这件事做成了，对你/公司最重要的意义是什么？你怎么判断它做得好还是不够好？」

当客户能用语言描述出「改善后的状态」，「改变真的有可能发生」才会在他心里生根。

## 第14章 履行承诺

### 🎯 核心主张
承诺不是达成交易，而是让客户真正理解「达到目标需要付出什么」，并且愿意为此行动。

### 📌 关键框架
**普通承诺 vs 共同承诺**：

普通承诺（只是分工）：
> 「我负责写项目大纲，乔伊负责指标，28日前完成。」

共同承诺（有深度的合作）：
> 「我建议让茱蒂也参与大纲撰写，她在银行领域有经验。比尔在销售部门，会对你的指标感兴趣。还有玛雅——这个任务是不是一个好机会，让你开始把一些工作委派给下属？」

**共同承诺的顾问是无法被替代的。**

### 🔧 可落地的交付物
**项目启动前的承诺质量检验**：
- [ ] 客户知道「做到什么程度」算完成了吗？
- [ ] 双方对时间节点的理解是一致的吗？
- [ ] 我有没有为了赢得项目而承诺了我做不到的事？
- [ ] 这个承诺是「我来做」还是「我们一起做」？（后者更强）

> 重要的事情说三遍：永远不要为了赢得项目而承诺做不到的事情。这不值得。这不值得。**这不值得。**

## 第二部分总结

### 信任等式——每次客户互动后的自检

| 维度 | 自评 | 具体行动 |
|------|------|---------|
| C 可信度 | /10 | 下次见面前做哪些准备？ |
| R 可靠度 | /10 | 上次承诺的小事兑现了吗？ |
| I 亲近感 | /10 | 有没有一件真实的事还没说出口？ |
| S 自我导向 | /10（越低越好） | 今天开会，我的关注点在哪里？ |

### 五阶段——卡点快速诊断

```
客户不开口    → 委托阶段有问题 → 我带来了什么值得他说的？
客户觉得没被理解 → 倾听阶段有问题 → 我在听他说的，还是我想听的？
方向不一致    → 界定/构想阶段 → 我们对问题/目标的理解一样吗？
同意了但不动  → 承诺阶段有问题 → 往前退两步，重新确认
```
