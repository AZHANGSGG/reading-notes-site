# 第三部分：践行信任关系（第15-22章）

> **本部分核心**：道理都懂，为什么做不到？怎么在真实场景中落地？

---

## 第15章 践行信任关系难在何处

### 🎯 核心主张
做不到的原因不是理解不了，而是本能在阻抗——我们天生趋向自我保护，而信任要求你先放下自我。

### 📌 关键框架
**14个常见借口与解构**：

| 借口 | 解构 |
|------|------|
| 这样做太冒险 | 谨小慎微同样让人不舒服。冒险是亲近感的前提 |
| 无法时刻关注别人 | 关注别人能给「自己」带来更多好处 |
| 机构文化是专业内容至上 | 最好的机构已经把信任当成安身立命之本 |
| 我怕显得无知 | 不懂装懂才会显得更无知 |
| 无法改变急于开口的习惯 | 习惯可以改变，需要具体的物理提醒 |
| 说真话太冒险 | 不说真话会使事情恶化，代价更大 |
| 可能侵犯客户隐私 | 你担心的其实是自己面对对方反应时的窘态 |
| 会损害专业内容的价值 | 完全相反——信任让专业内容发挥更大作用 |
| 方法太教条 | 从实用主义判断：这套方法有效吗？ |
| 太花时间 | 基于两个错误假设：客户不为信任买单、面谈必须当期回本 |
| 客户只要我做眼前的事 | 你没尝试改变，就假定了这是固定的 |
| 没把握就不应该表态 | 不表态才是最不专业的行为 |
| 表态错了会失去信誉 | 固守错误的立场才真正危险 |
| 要保持谦卑太难了 | 谦卑不是懦弱，承认他人的强大不损害自己的强大 |

**两种风险的不对等**：
- 做了不该做的事 → 容易被发现和纠正
- **应该做的事没有做** → 往往被忽视，但代价更大、更持久

大多数顾问为了避免前者，在不知不觉中承担了后者。

### 🔧 可落地的交付物
**「应该做但没做」检查**（每月一次，对照最近一个月的客户互动）：
- 有哪些话，我觉得应该说但没说？（直言不讳的缺失）
- 有哪些主动联系，我觉得应该做但没做？（关系维护的缺失）
- 有哪些承诺，我觉得应该不做但还是答应了？（期望管理的缺失）

> 「应该做的事没做，往往是最危险的风险——它不容易被发现，代价却最为深远。」

## 第16章 定义不同类型的客户

### 🎯 核心主张
有效的顾问工作从理解「这个客户与其他客户有何不同」开始——用同一套方式对待所有人，是信任建立的最大障碍之一。

### 📌 关键框架
**九种棘手客户及应对速查**：

| 类型 | 核心特征 | 应对策略 |
|------|---------|---------|
| 「我只在乎事实」型 | 强势、直接、质疑一切 | 用他的语言，澄清并确认他重视的价值 |
| 「我会再联系你」型 | 保守、害怕犯错 | 提前预判，准备简洁总结，给他时间 |
| 「你是专家你做主」型 | 已放弃控制或很挫败 | 提出假设和选择，带他参与进来 |
| 「让我来处理」型 | 不信任你，或谁都不信任 | 在私下里讨论他的安全感问题 |
| 「再梳理一遍」型 | 以细节为导向，控制欲强 | 把大局变得同样可控 |
| 「你不了解情况」型 | 强调自身独特性 | 虚心求教，承认不同，不争论 |
| 「敌人的敌人是朋友」型 | 喜欢办公室政治 | 把政治当作研究对象，私下坦诚讨论 |
| 「就像……你懂的」型 | 表达能力弱，但洞察力强 | 帮他找到表达方式，用书面选项引导 |
| 「哦对了还有一件事」型 | 总是错过重要会议/信息 | 直言不讳，私下要求澄清 |

### 🔧 可落地的交付物
**客户类型识别 + 调整策略**（接触新客户后填写）：
1. 这个客户最像上面哪种类型？
2. 他的核心恐惧或核心需求是什么？（比「他很难搞」更有用的判断）
3. 我需要在哪里调整自己的风格去适应他？

## 第17章 科伦坡中尉的秘密武器

### 🎯 核心主张
最有效的顾问风格不是展示权威，而是**稀释期望**——让客户放松，把关注点完全转向他们。

### 📌 关键框架
科伦坡中尉：不修边幅、迷迷糊糊、开旧车——让犯罪嫌疑人放松警惕。但这全是精心设计的。

对应到顾问工作：
- 客户对「专家」有天生的戒备（你知道得太多 → 他显得无知 → 他防御）
- 稀释权威感，让客户主动开口
- 让顾问无法采用科伦坡方式的，恰恰是顾问们最钟爱的东西：证书、头衔、刻意炫耀的专业资历

### 🔧 可落地的交付物
**「降低防御」话术清单**（在客户明显有防御时使用）：
- 「这只是我的初步判断，你可能有我没考虑到的信息……」
- 「我在这个行业的经验不如你，但从外部视角看……」
- 「你比我更了解内部情况，我只是想提供一个不同的角度……」

> 「最大的敌人就是我们自己。」——顾问展示权威的冲动，往往正是客户建立防御的原因。

## 第18-20章 营销、维系与交叉销售

### 🎯 核心主张（三章合并）
- **第18章**：营销和服务是同一枚硬币的两面——做好服务本身就是最好的营销
- **第19章**：项目执行中是建立信任的黄金期，大多数顾问只关注交付，错过了深化关系的机会
- **第20章**：项目结束后才是关系真正开始的时候
- **第21章**：交叉销售是信任的副产品，没有信任就想交叉销售，是缘木求鱼

### 🔧 可落地的交付物
**四个场景的行动清单**：

| 场景 | 核心任务 | 立刻可做的一件事 |
|------|---------|---------------|
| 赢得新客户 | 带着洞察去见面，不是资历清单 | 见面前用30分钟研究他最近的业务动态 |
| 项目进行中 | 每次接触都深化关系 | 下次项目会议主动分享一条与项目无关但他会感兴趣的信息 |
| 项目结束后 | 主动保持联系 | 项目结束后2周，发一条无商业目的的联系：「最近看到这个，想到你……」 |
| 交叉销售 | 在信任基础上自然延伸 | 先问「你现在最头疼的问题是什么」，再判断是否有你能帮的 |

**瑞贝卡的教训**：项目结束后没有主动联系，理由是「担心显得唐突」。一年后客户电话来，带着埋怨：「你为什么不再打电话给我们？」——**主动联系的风险远远小于沉默的代价。**

## 第22章 速效清单

### 🎯 核心主张
如果只记住一件事：**把关注点从自己身上移开，放到客户身上**。

### 🔧 可落地的交付物
**可信度（C）——立刻可做**：
- 当不知道答案时，直接说不知道，不要含糊带过
- 见面前研究他的行业，带一个他关心的问题去
- 热爱你的专业，别人看得出来

**可靠度（R）——立刻可做**：
- 在小事上做出具体承诺，不露声色地兑现
- 对任何延误或变更，在对方问之前主动告知

**亲近感（I）——立刻可做**：
- 在没有特殊意义的日子主动联系一个老客户
- 下次开会时，在正式议题开始前，先花两分钟问问他个人最近的状态

**自我导向（S）——立刻可做**：
- 从小处开始：选一个现在可以改变的行为，只改一个，做两周，观察结果
- 每次客户互动结束后问：「刚才那次，我的注意力有多少在客户身上，有多少在自己身上？」

## 第三部分总结

| 场景 | 核心任务 | 常见错误 | 立刻可做的一件事 |
|------|---------|---------|---------------|
| 面对借口 | 识别真实的恐惧 | 合理化「做不到」 | 选14个借口里最像自己的一个，直面它 |
| 面对难搞客户 | 理解他的核心需求 | 用同一套方式对待所有人 | 识别他是哪种类型，调整一个具体行为 |
| 项目进行中 | 每次接触都深化关系 | 只关注交付 | 下次会议主动问一句非项目相关的问题 |
| 项目结束后 | 主动保持联系 | 等客户有需求才出现 | 项目结束2周内发一条无商业目的的联系 |

### 全书终极提醒

> 「应该做的事没做，往往是最危险的风险——它不容易被发现，代价却最为深远。」

> 「勇气，是被信任的顾问与普通顾问之间最真实的分水岭。」
